Kafka (Event Bus)
+-------------------------------------------------------+
| - topics: cluster-events, logs.*                       |
| - core-api produces events (job.requested... )         |
| - core-api consumes events (consumer -> EventHandler)  |
+-------------------------------------------------------+

Logging / Monitoring stack (observability)
+---------+       +------------+          +------------+
|  Logs   | ----> | log-consumer| ------> |   Loki     |
+---------+       +------------+          +------------+
                             ^                        |
                             |                        |
                          Kafka (logs.*)             Grafana
                                                      (Dashboards & Explore)

Prometheus (Metrics)
+-----------------------------------------------------+
| core-api exposes /metrics -> Prometheus scrapes      |
| Prometheus -> Grafana (dashboards & alerting rules) |
+-----------------------------------------------------+

Notes:
- Docker Compose used to orchestrate core-api, MySQL, Redis, Kafka, Prometheus, Grafana, Loki, log-consumer for local dev.
- Frontend talks to core-api via REST; embeds Grafana dashboards for monitoring.
- log-consumer enriches logs with labels (job_id, trace_id) and pushes to Loki for easy correlation with metrics in Grafana.
